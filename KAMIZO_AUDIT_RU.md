## 🔴 Критические логические пробелы (выглядит готовым, но на деле сломано)

1. **Мульти-тенантность: несоответствие схемы** — бэкенд пишет `tenant_id` в 41 таблицу, где этой колонки нет. INSERT-ы либо молча падают, либо данные текут между тенантами. (`cloudflare/schema.sql` — таблицы: users, buildings, apartments, requests, vehicles, chat_channels, chat_messages, meetings, marketplace_products, marketplace_orders, notifications и ещё 30)

2. **Дедлайн голосования собраний не проверяется** — `POST /api/meetings/:meetingId/agenda/:agendaItemId/vote` (index.ts ~строка 10353) проверяет `meeting.status !== 'voting_open'`, но НЕ проверяет `voting_deadline > NOW()`. Голоса принимаются после дедлайна, если админ забыл закрыть.

3. **Голоса изменяемы (UPDATE, а не INSERT-only)** — `cloudflare/src/index.ts` строка ~10431: `UPDATE meeting_vote_records SET choice = ?`. Оригинальный голос перезаписывается. Нарушает неизменяемость аудита. Хеш голоса есть, но не предотвращает модификацию.

4. **В маркетплейсе нет оплаты** — заказы создаются через `POST /api/marketplace/orders` (строка ~13981) без верификации оплаты. Параметр `payment_method` принимается, но не обрабатывается. Выручку собрать нельзя.

5. **Показания счётчиков принимают отрицательный расход** — нет проверки `текущее_показание >= предыдущее_показание` при приёме показаний. Нет окна приёма показаний (например, 1-10 числа месяца).

6. **Нет компонента ProtectedRoute** — роуты в `Layout.tsx` без guard'ов на уровне маршрутов. Разлогиненный пользователь может оставаться на `/admin` до перезагрузки страницы. Проверки ролей только внутри отдельных компонентов.

7. **Токен аутентификации = голый user.id** — `authStore.ts` хранит `user.id` как токен. Не JWT. Если user ID предсказуемы/последовательны, любой может выдать себя за другого через localStorage.

8. **Rate limiter пропускает при ошибке** — `cloudflare/src/middleware/rateLimit.ts` строки 54-57: при ошибке KV возвращает `allowed: true` с 99 оставшимися запросами. Атакующий может эксплуатировать деградацию KV для брутфорса.

9. **Захардкоженный ключ шифрования в Git** — `cloudflare/wrangler.toml` строка 17: `ENCRYPTION_KEY = "K4m1z0-S3cur3-Encrypt10n-2026!"` в репозитории.

10. **Обратимое хранение паролей** — колонка `password_plain TEXT` хранит AES-GCM зашифрованные пароли (не bcrypt-хеш). В сочетании с открытым ключом шифрования — все учётные данные скомпрометированы.

11. **SQL-инъекция в LIKE-паттерне уведомлений** — `cloudflare/src/index.ts` строка ~12786: тег уведомления подставляется напрямую в LIKE вместо параметризованного запроса.

12. **Не проверяется max_uses гостевого доступа** — `POST /api/guest-codes/:id/use` увеличивает счётчик использований, но не проверяет `current_uses >= max_uses` перед разрешением входа.

## 🟡 Неполные цепочки (частично реализованные фичи)

1. **Оплата/Финансы** — отображение задолженности есть (`GET /api/accounts/debtors`), но нет эндпоинта записи оплаты, генерации квитанций, сверки платёж-счёт.

2. **Автоматический расчёт расхода по счётчикам** — показания хранятся, но нет автоматического `расход = текущее - предыдущее`. Не проверяется дата поверки счётчика.

3. **Вложения в чате** — чат только текстовый. Нет таблицы `chat_message_attachments` или эндпоинта загрузки. Обмен медиа невозможен.

4. **Голосование по доверенности** — закон РУз может требовать доверенности на голосование. Нет таблиц proxy_voter или механизма делегирования.

5. **Блокировка периода рейтинга УК** — жители могут менять рейтинг в любое время в течение месяца. Нет блокировки после первой отправки или окончания месяца.

6. **Аудит-лог активности** — `ActivityLogPage.tsx` формирует лог только из статусов заявок — нет системного аудита действий пользователей, изменений данных или доступа.

## 🟢 Полностью работающие фичи

1. **Жизненный цикл заявок** — полная цепочка: создание (авто-номер по филиалу+категории) → назначение → в работе → завершена → оценка. Оптимистичный UI с откатом. Push-уведомления при создании и назначении. Видимость по ролям.

2. **Создание собраний и голосование (базовый поток)** — вес голоса = площадь квартиры (кв.м) по закону РУз. Кворум = 50%+ площади здания. UNIQUE-ограничение предотвращает дублирование. Генерация протокола. Пересмотр голоса.

3. **Гостевой доступ и QR-коды** — самодостаточные GAPASS-токены (base64 JSON с истечением срока). Охранник валидирует офлайн. Типы: одноразовый, день, неделя, месяц, произвольный. Логирует вход/выход.

4. **Чат/Мессенджер (базовый)** — реалтайм через WebSocket. Типы каналов: uk_general, building, private_support. Курсорная пагинация. Отслеживание прочитанных.

5. **Управление транспортом** — полный CRUD с поиском по номеру. Типы: авто, мото, велосипед. Валидация регионов УЗ.

6. **Объявления** — полный CRUD с отслеживанием просмотров. Приоритеты. Таргетинг по зданию/подъезду/этажу/филиалу. Push при создании.

7. **Профиль жителя и договор** — редактирование, смена пароля, выбор языка, QR-код договора, подпись.

8. **Наряд-заказы** — авто-номер `НР-YYYY-NNN`. CRUD со статусами. Назначение исполнителей.

9. **Рейтинг удовлетворённости УК** — месячный период. UPSERT предотвращает дублирование. 4 метрики (общая, чистота, оперативность, коммуникация). 6-месячная сводка для админа.
