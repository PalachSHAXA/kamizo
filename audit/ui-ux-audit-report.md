# Kamizo — UI/UX Аудит

**Платформа:** https://demo.kamizo.uz
**Дата:** 2026-03-11
**Версия:** PWA (React + Vite)
**Проверенные роли:** Житель, Менеджер
**Устройства:** Mobile (390px), Tablet (768px), Desktop (1280px)

## 2. Сильные стороны

### 2.1 Цветовая система
- Тёплая оранжевая палитра (#FF6B00 → #FFA500) создаёт дружелюбное ощущение
- Светлый кремовый фон (#FFF8F0) вместо холодного белого — приятно для глаз
- Цветовое кодирование статусов заявок (зелёный = выполнена, красный = отменена, жёлтый = новая) — интуитивно

### 2.2 Mobile-first подход
- Карточный дизайн на дашборде отлично работает на мобильном
- Гамбургер-меню с badge-уведомлениями
- Bottom sheet-стиль страниц (закруглённые углы карточек)
- Хорошее использование пространства на мобильном

### 2.3 Функциональная полнота
- Маркетплейс с полноценным каталогом товаров
- QR-код для гостевого доступа — умное решение
- Поиск автомобиля по узбекскому номерному знаку — великолепный UX
- Голосования со встроенным кворумом
- Договор с QR + электронная подпись

### 2.4 Десктопная версия
- Хорошая боковая навигация с группировкой по категориям (Операции, Доступ и имущество, Информация)
- Шапка с часами, поиском и профилем
- Таблоидный layout маркетплейса на десктопе с сеткой карточек

## 3. Критические проблемы (Priority 1 — Исправить срочно)

### 3.1 Баннер "Включить уведомления?" закрывает контент
**Где:** Каждая страница, каждая роль, каждое устройство
**Проблема:** Sticky баннер снизу занимает ~120px высоты и перекрывает контент/кнопки на КАЖДОЙ странице. На скриншотах видно что он закрывает:
- Графики на дашборде менеджера
- Записи в списке заявок
- Содержимое чата
- Карточки на гостевом доступе

**Решение:**
- Показывать баннер ОДИН раз при первом визите, затем скрывать навсегда
- Если отклонено — не показывать минимум 30 дней
- Уменьшить высоту баннера (сейчас слишком много текста)
- Не показывать на внутренних страницах (только на дашборде)

### 3.2 Пустой контент после логина менеджера
**Где:** Desktop, после логина менеджера
**Проблема:** На скриншоте `login-manager-result.png` — после логина менеджер видит пустую страницу с только боковой навигацией. Нет автоматического перехода на дашборд.
**Решение:** Автоматически редиректить на `/dashboard` после успешного логина.

### 3.3 Категории маркетплейса обрезаются на мобильном
**Где:** Маркетплейс, мобильная версия
**Проблема:** Горизонтальный скролл категорий — пользователь не видит все категории, нет индикатора что можно скроллить вправо.
**Решение:** Добавить fade-эффект справа или стрелку "ещё", либо сделать chips с wrap.

## 4. Важные проблемы (Priority 2 — Исправить в ближайшем релизе)

### 4.1 Информационная перегрузка дашборда жителя
**Где:** Дашборд жителя (mobile)
**Проблема:** Слишком много разнородных блоков на одном экране:
- Приветствие + адрес
- Статус заявки
- Действия (Вызвать мастера)
- Чат + Гости
- Магазин
- Найти авто + Контакты
- Оплата / Квитанции / Счета
- Новости дома
- Нижнее меню (Собрания, Оценить, Договор, Заявки)

Это 10+ различных секций, и пользователю приходится долго скроллить.

**Решение:**
- Сгруппировать по приоритету: Основное (заявки, оплата) → Сервисы (мастер, гости, авто) → Информация (новости, объявления)
- Скрыть редко используемые действия в "Ещё"
- Или использовать табы: Главная / Сервисы / Оплата

### 4.2 Нижнее меню жителя — нелогичный набор пунктов
**Где:** Footer navigation жителя
**Проблема:** Пункты "Собрания", "Оценить", "Договор", "Заявки" — странный набор для постоянного нижнего меню. "Договор" вряд ли нужен ежедневно, а "Чат" (ежедневный) — нет в нижнем меню.
**Решение:** Нижнее меню должно содержать:
- Главная
- Заявки
- Чат
- Уведомления
- Ещё (профиль, договор, оценки, собрания)

### 4.3 Дублирование "свыс" в объявлениях
**Где:** Объявления (mobile)
**Проблема:** Два одинаковых объявления "свыс / высыс" с одинаковым изображением — выглядит как баг или тестовые данные, просочившиеся в демо.
**Решение:** Проверить демо-данные, удалить дубликаты.

### 4.4 Контраст текста на карточках статусов
**Где:** Дашборд менеджера, карточки "Новые/В работе/Ожидают/За неделю"
**Проблема:** Белый текст на светлых цветных фонах (особенно жёлтый и зелёный) — низкий контраст. Не проходит WCAG AA.
**Решение:** Использовать тёмный текст на светлых фонах или затемнить фоны карточек.

### 4.5 Временные данные на рабочих нарядах
**Где:** Наряды на работы (manager mobile)
**Проблема:** Даты нарядов из 2024 года ("25.12.2024", "26.12.2024", "15.10.2024") в демо 2026 года — выглядит неактуально и вызывает недоверие к демо.
**Решение:** Обновить seed-данные с актуальными датами.

## 5. Улучшения UX (Priority 3 — Запланировать)

### 5.1 Отсутствие пустых состояний (Empty States)
**Где:** Заметки, некоторые списки
**Что хорошо:** Заметки имеют отличное пустое состояние с CTA "Создать первую заметку"
**Что плохо:** Другие пустые списки показывают просто пустое пространство без объяснения
**Решение:** Добавить иллюстрации и CTA для всех пустых состояний

### 5.2 Отсутствие подтверждения действий
**Где:** Удаление автомобиля (иконка корзины), отмена заявки
**Проблема:** Деструктивные действия должны требовать подтверждения
**Решение:** Добавить confirmation dialog перед удалением

### 5.3 Чат — нет индикатора набора текста
**Где:** Чат с УК
**Проблема:** Нет "печатает...", нет прочитано/доставлено (только время отправки)
**Решение:** Добавить typing indicator и статусы сообщений (отправлено ✓ / доставлено ✓✓ / прочитано 🟢)

### 5.4 Мониторинг — текст "Database", "Cache", "WebSocket" на английском
**Где:** Мониторинг (manager mobile)
**Проблема:** Техническая страница смешивает русский и английский. "Статус: Здорово" — неестественный перевод "Healthy" (лучше "Работает" или "Норма").
**Решение:** Либо полностью на русском ("БД", "Кэш", "Вебсокет"), либо оставить англоязычные технические термины, но исправить "Здорово" → "Норма".

### 5.5 Профиль жителя — перегружен
**Где:** Профиль (mobile)
**Проблема:** На одной странице: личные данные, язык, договор с QR, безопасность, инструкция установки PWA, уведомления, данные для входа. Очень длинный скролл.
**Решение:** Разбить на секции/табы: Профиль / Безопасность / Приложение

### 5.6 Гостевой доступ — мелкий текст в табах
**Где:** Гостевой доступ (mobile)
**Проблема:** Табы "Активные / Использованные / Истекшие" обрезаются, текст мелкий
**Решение:** Использовать иконки + короткие подписи или горизонтальный скролл с chip-стилем

### 5.7 Маркетплейс — нет отзывов и рейтингов товаров
**Где:** Маркетплейс
**Проблема:** Карточки товаров имеют только цену и название. Нет рейтинга, количества продаж, отзывов.
**Решение:** Добавить звёзды рейтинга и количество отзывов на карточку

### 5.8 Поиск авто — великолепный ввод номера, но нет истории
**Где:** Поиск автомобилей (manager)
**Проблема:** Отличный UX ввода номера с узбекским форматом, но нет истории последних поисков
**Решение:** Добавить "Последние поиски" под полем ввода

## 6. Адаптивность (Responsive)

### 6.1 Mobile (390px) — Оценка: 8.5/10
- Отлично оптимизирован, карточный дизайн
- Хорошее использование пространства
- Гамбургер-меню работает
- **Минус:** Баннер уведомлений перекрывает контент

### 6.2 Tablet (768px) — Оценка: 7/10
- По сути тот же mobile layout, просто растянутый
- Не использует дополнительное пространство для двухколоночной сетки
- На дашборде менеджера появляется "По категориям" рядом с графиком — хорошо
- **Минус:** Карточки статусов растягиваются на всю ширину — лучше было бы 2×2

### 6.3 Desktop (1280px) — Оценка: 7.5/10
- Боковая навигация хорошо структурирована
- Маркетплейс отлично использует сетку
- Часы в шапке — приятная деталь
- **Минус:** Дашборд жителя не полностью использует ширину
- **Минус:** Много пустого пространства справа на некоторых страницах

## 7. Консистентность дизайна

### 7.1 Хорошо
- Единый стиль карточек с закруглёнными углами (12-16px border-radius)
- Оранжевая палитра используется последовательно
- Иконки из одного набора (outline-стиль)
- Badge-уведомления единообразны (красный кружок с числом)

### 7.2 Проблемы
- **Кнопки:** Разные стили на разных страницах — "Проголосовать" (зелёный), "+ Создать наряд" (оранжевый заливка), "+ Добавить" (оранжевый outline). Нужна единая система кнопок.
- **Карточки статусов:** На дашборде менеджера — цветные фоны с иконками, а на заявках — outline badges. Разный визуальный язык.
- **Тени:** Некоторые карточки с тенью, некоторые без — нет единого правила.

## 8. Рекомендации по улучшению (Quick Wins)

### Можно сделать за 1-2 дня:
1. **Убрать навязчивый баннер уведомлений** — показывать 1 раз, запоминать в localStorage
2. **Исправить редирект после логина** — всегда перенаправлять на дашборд
3. **Обновить демо-данные** — актуальные даты, убрать дубли объявлений
4. **Исправить "Здорово"** → "Норма" в мониторинге

### Можно сделать за 1 неделю:
5. **Пересмотреть нижнее меню жителя** — Главная, Заявки, Чат, Уведомления, Ещё
6. **Добавить fade-индикатор** на горизонтальные скроллы (категории маркетплейса)
7. **Добавить confirmation dialogs** для деструктивных действий
8. **Улучшить пустые состояния** на всех страницах

### На будущее:

10. **Pull-to-refresh** на мобильных списках

## 9. Конкуренотоспособность

Для PropTech/ЖКХ-решения в Узбекистане — приложение на **высоком уровне**:
- Функциональность шире, чем у большинства аналогов
- PWA вместо нативного приложения — правильное решение для рынка
- Многоязычность (РУ/УЗ) — обязательна и реализована
- Поиск по номерному знаку — уникальная фича
- Маркетплейс — серьёзное конкурентное преимущество

**Основной вектор роста:** Полировка UX (микроанимации, loading states, transitions) и устранение мелких раздражителей (баннер уведомлений, пустые экраны).
