# 1. ОБЗОР ПРОЕКТА

## 1.1 Что это?
**UK CRM** — это революционная экосистема управления жилищным фондом, созданная специально для управляющих компаний Узбекистана. Это не просто CRM — это цифровой мост между УК, сотрудниками и жителями, который превращает хаос в порядок, а недовольство — в лояльность.

## 1.2 Миссия
> "Сделать жизнь в многоквартирном доме комфортной, прозрачной и современной"

## 1.3 Целевая аудитория
- **Управляющие компании** — полная автоматизация и контроль
- **Жители и собственники** — удобство и участие в управлении
- **Сотрудники** — эффективность и мотивация
- **Арендаторы** — быстрая связь с УК
- **Рекламодатели** — доступ к платежеспособной аудитории

# 2. ДИЗАЙН И ВИЗУАЛЬНЫЙ СТИЛЬ

## 2.1 Концепция — "Солнечный Glass-morphism"

### Философия дизайна
Желтый цвет выбран не случайно — это цвет солнца, тепла и доверия. В сочетании с эффектом "матового стекла" (glass-morphism) создается ощущение:
- **Премиальности** — как интерфейс банковского приложения
- **Легкости** — полупрозрачные элементы "парят" над фоном
- **Современности** — тренд 2024-2026 годов в UI/UX
- **Узнаваемости** — желтый = УК, коммунальные услуги

### Цветовая палитра
```
ОСНОВНЫЕ ЦВЕТА:
┌─────────────────────────────────────────────────────────────┐
│  🟡 #FFE500  │  Primary     │  Кнопки, акценты, активные   │
│  🟡 #FFED4D  │  Hover       │  Наведение на элементы       │
│  ⬜ Gradient │  Background  │  #fefefe → #fff9e6 → #fffef5 │
│  ⬛ #1a1a1a  │  Text        │  Основной текст              │

СТАТУСЫ:
```

### Glass-morphism эффекты
```css
/* Карточка с эффектом матового стекла */
.glass-card {
  background: rgba(255, 255, 255, 0.4);      /* 40% белого */
  backdrop-filter: blur(12px);               /* Размытие фона */
  -webkit-backdrop-filter: blur(12px);       /* Safari support */
  border: 1px solid rgba(255, 255, 255, 0.5);/* Светлая граница */
  border-radius: 16px;                       /* Мягкие углы */
  box-shadow: 0 8px 32px rgba(0,0,0,0.08);   /* Парящая тень */
}

/* При наведении — эффект "всплытия" */
.glass-card:hover {
  background: rgba(255, 255, 255, 0.5);
  transform: translateY(-2px);               /* Приподнимается */
  box-shadow: 0 16px 48px rgba(0,0,0,0.12);  /* Тень глубже */
}
```

# 5. ОСНОВНЫЕ МОДУЛИ СИСТЕМЫ

## 5.1 📝 СИСТЕМА ЗАЯВОК (Request Management)

### Ценность модуля
> **До:** Житель звонит в УК → ждет на линии 10 минут → диспетчер записывает в журнал → теряется бумажка → никто не приходит → конфликт
>
> **После:** Житель за 30 секунд создает заявку → автоматическое назначение → push исполнителю → отслеживание статуса → оценка качества

### Жизненный цикл заявки

```
    ┌─────────┐
    │   NEW   │  Житель создал заявку
    │  Новая  │
    └────┬────┘
         │ Автоматически или менеджером
         ▼
    ┌─────────┐
    │ASSIGNED │  Назначен исполнитель
    │Назначена│
    └────┬────┘
         │ Исполнитель принял
         ▼
    ┌─────────┐
    │ACCEPTED │  Исполнитель едет
    │ Принята │
    └────┬────┘
         │ Исполнитель на месте
         ▼
    ┌──────────┐     ┌────────┐
    │IN_PROGRESS│◀───│ PAUSED │  Можно ставить на паузу
    │ В работе ├────▶│ Пауза  │  (обед, запчасти)
    └────┬─────┘     └────────┘
         │ Работа завершена
         ▼
    ┌──────────────┐
    │PENDING_REVIEW│  Ожидает проверки менеджера
    │  На проверке │
    └────┬─────────┘

         ▼

    └─────────┘
```

### Приоритеты заявок
| Приоритет | Цвет | SLA | Примеры |
|-----------|------|-----|---------|
| 🟢 LOW | Зеленый | 7 дней | Замена лампочки в подъезде, покраска |
| 🟡 MEDIUM | Желтый | 3 дня | Течет кран, скрипит дверь |
| 🟠 HIGH | Оранжевый | 24 часа | Не работает лифт, нет горячей воды |
| 🔴 URGENT | Красный | 4 часа | Прорыв трубы, короткое замыкание, пожар |

### Умное назначение
Система автоматически выбирает исполнителя по:
1. **Специализация** — сантехник на сантехнику
2. **Загруженность** — кто менее занят
3. **Рейтинг** — приоритет лучшим
4. **Геолокация** — кто ближе к объекту
5. **История** — кто уже работал с этим жителем

### Перенос заявки (Reschedule)
Если исполнитель не может прийти в назначенное время:
1. Нажимает "Запросить перенос"
2. Указывает причину и новое время
3. Житель получает уведомление
4. Житель подтверждает или предлагает другое время
5. Заявка переносится с сохранением истории

### Оценка качества
После завершения заявки житель может:
- Поставить оценку 1-5 звезд
- Написать комментарий
- Оценка влияет на рейтинг исполнителя

## 5.2 🗳️ СОБРАНИЯ СОБСТВЕННИКОВ (Meetings/OSS)

### Юридическая сила
Система соответствует законодательству Республики Узбекистан:
- Голосование по площади квартиры (м²)
- Кворум — 50% + 1 от общей площади
- OTP-верификация каждого голоса
- Неизменяемый аудит

### Полный цикл собрания

```
┌─────────────────────────────────────────────────────────────┐
│                    ЖИЗНЕННЫЙ ЦИКЛ OSS                       │
├─────────────────────────────────────────────────────────────┤

 1. DRAFT                    Организатор создает черновик
    ↓                        • Тема собрания
                             • Вопросы повестки
                             • Документы

 2. PENDING_MODERATION       Администратор проверяет
    ↓                        • Корректность формулировок
                             • Соответствие законодательству

 3. SCHEDULE_POLL            Жители голосуют за дату
    ↓                        • Предложено 3-5 вариантов
                             • Каждый выбирает удобную

 4. SCHEDULE_CONFIRMED       Дата определена
    ↓                        • Выбрана по большинству
                             • Уведомления всем жителям

 5. VOTING                   Голосование по повестке
    ↓                        • Каждый вопрос отдельно
                             • За / Против / Воздержался
                             • OTP-подтверждение

 6. RESULTS                  Подсчет результатов
    ↓                        • Автоматический подсчет
                             • Учет веса голосов (м²)

 7. PROTOCOL                 Генерация протокола
    ↓                        • Автоматический Word-документ
                             • Все данные заполнены

 8. APPROVED                 Подписание и утверждение
                             • Электронные подписи
                             • Юридическая сила

└─────────────────────────────────────────────────────────────┘
```

### Вес голоса
```

Общая площадь дома: 10,000 м²
Кворум: 5,001 м² (50% + 1)

Квартира 45 (Иванов): 65 м²
Квартира 46 (Петров): 120 м² (2 собственника по 50%)
Квартира 47 (ООО "Рога"): 200 м²

Голосование по вопросу "Ремонт подъезда":
```
