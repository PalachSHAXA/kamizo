# Регуляторные требования для Узбекистана

## Закон ЗРУ-581 "Об управлении многоквартирными домами"

### Основные положения

**Регистрация УК:**
- С декабря 2024 года УК обязаны регистрироваться в электронном реестре Минстроя
- Регистрация через портал my.gov.uz
- Стоимость: 187 500 сумов (168 750 через ЕПИГУ со скидкой 10%)

### Требования к CRM

**Обязательная интеграция:**
- Возможность выгрузки данных для реестра Минстроя
- Хранение договоров управления
- Учет решений общих собраний

**Отчетность:**
- Формирование отчетов по форме Минстроя
- Данные о выполненных работах
- Финансовая отчетность по ЖК

## Закон ЗРУ-547 "О персональных данных"

### Критические требования

**Локализация данных:**
```
ВСЕ персональные данные граждан Узбекистана
ДОЛЖНЫ храниться на серверах внутри страны
```

**Определение персональных данных:**
- ФИО
- Адрес проживания
- Номер телефона
- Email
- Данные документов
- Фотографии
- Биометрия

### Технические требования

**Хостинг:**
- Серверы на территории Узбекистана
- Рекомендуемые провайдеры:
  - Uztelecom дата-центры
  - Облачные провайдеры с локальным размещением

**Регистрация базы данных:**
- Обязательная регистрация в Государственном реестре баз персональных данных
- Уполномоченный орган: Государственная инспекция по контролю в сфере информатизации

### Требования к безопасности

**Технические меры:**
- Шифрование данных при хранении (AES-256)
- Шифрование при передаче (TLS 1.3)
- Журналирование доступа
- Резервное копирование

**Организационные меры:**
- Политика обработки персональных данных
- Согласие на обработку от пользователей
- Назначение ответственного лица

### Права субъектов данных

**Пользователи имеют право на:**
- Доступ к своим данным
- Исправление неточных данных
- Удаление данных
- Ограничение обработки
- Переносимость данных

**Реализация в CRM:**
- Профиль пользователя с возможностью редактирования
- Экспорт данных в читаемом формате
- Функция удаления аккаунта
- Политика конфиденциальности

## Требования к платежам

### Payme

**Регуляторные требования:**
- Договор с Payme как мерчант
- Соответствие PCI DSS (на стороне Payme)
- Не хранить данные карт на своей стороне

**Интеграция:**
```
Merchant ID: получить от Payme
Secret Key: безопасное хранение
Webhook URL: для уведомлений о платежах
```

### Click

**Регуляторные требования:**
- Регистрация как мерчант Click
- Подписание договора
- Тестирование интеграции

**Интеграция:**
```
Service ID: получить от Click
Merchant ID: получить от Click
Secret Key: безопасное хранение
```

### Узбекские карты

**UZCARD:**
- Национальная платежная система
- 21+ банков-партнеров
- Интеграция через Payme/Click

**HUMO:**
- Альтернативная платежная система
- HUMOPay мобильный кошелек
- Бесконтактные платежи

## Требования к Telegram-боту

### Telegram ToS

**Ограничения:**
- Не спамить пользователей
- Не собирать данные без согласия
- Соблюдать Telegram Bot API ToS

### Реализация

**Согласие пользователя:**
- При первом запуске бота - согласие на обработку данных
- Возможность отписаться от уведомлений
- Команда /delete для удаления данных

**Безопасность:**
- Верификация пользователя (номер телефона или код)
- Не передавать конфиденциальные данные в открытом виде
- Логирование действий

## Мультиязычность

### Требования

**Обязательные языки:**
- Узбекский (латиница) - uz-Latn
- Узбекский (кириллица) - uz-Cyrl
- Русский - ru

**Реализация:**
- Автоопределение языка по настройкам Telegram/браузера
- Возможность смены языка в настройках
- Все UI элементы локализованы
- Даты и числа в локальном формате
