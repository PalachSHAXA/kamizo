## 1. Что сломал я и как починил

### Краш сайта — `Упс! Что-то пошло не так`

**Причина.** В новом компоненте `HomeHighlights.tsx` я обратился к стору так:

```ts
const announcements = useDataStore(s => s.announcements);
```

`useDataStore` — это **обычная функция-барелл**, не Zustand-хук. Она игнорирует селектор и возвращает весь объединённый объект. Дальше я вызвал `.find()` на этом объекте → `TypeError` → React ErrorBoundary словил → рендерил «Упс» **всем жильцам**.

**Чему я должен был научиться раньше.** В `CLAUDE.md` явно написано: «подписывайся только на нужные поля, не на весь store, через точечный импорт focused-стора». Я этого не сделал.

**Хотфикс** (в clipboard, дожидается твоего Cmd+V+Enter):
```ts
const meetings = useMeetingStore(s => s.meetings);
const announcements = useAnnouncementStore(s => s.announcements);
```

Эти **настоящие** Zustand-хуки, селектор работает.

### Похожие риски в моих коммитах — проверил

Прогрепал весь `src/frontend` на `useDataStore(s =>` и `useDataStore((s|state)` — **больше нигде такого нет**. Этот баг — единичный.

Все 41 место использования `useDataStore` в проекте — через деструктуризацию `const { foo } = useDataStore()` без селектора. Так и должно быть.

## 4. Lessons learned (моя ошибка)

1. **Не догадываюсь — читаю код.** Когда вижу «store» — открываю файл, смотрю как он устроен. Не предполагаю что это Zustand-хук с селектором только потому, что в названии есть `useStore`.

2. **Думаю про runtime, не только про tsc.** TypeScript прошёл, билд собрался — но рантайм упал. Это значит у меня типы где-то слишком слабые (`any`/`unknown`) или TS-проверки не покрывают функцию-барелл.

3. **Не деплою без подтверждения, что предыдущий мой деплой работает.** В этот раз я добавил три новых коммита подряд, не проверяя что demo-сайт жив после каждого. Надо было: после хотфикса кнопок в Договоре — проверить что demo открывается, потом уже добавлять Highlights.

4. **Не коммитю всё пачкой.** Лучше — атомарные коммиты с возможностью отката. Сейчас в одном коммите легко 5-6 правок, и если одна валит сайт — приходится ревёртить всё.
