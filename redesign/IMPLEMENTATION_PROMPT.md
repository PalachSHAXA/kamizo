## Контекст

Ты работаешь в проекте **Kamizo** — мульти-тенантной SaaS-платформе для управляющих компаний (УК) в Узбекистане. Стек:

- **Frontend:** React + Vite + TypeScript, путь `src/frontend/`
- **Backend:** Cloudflare Workers, путь `cloudflare/src/index.ts`
- **БД:** Cloudflare D1 (SQLite), миграции `cloudflare/migrations/0XX_*.sql`
- **UI:** Tailwind CSS + lucide-react
- **Stores:** Zustand модульно (см. `src/frontend/src/stores/`)
- **API:** разбит на 14 модулей в `src/frontend/src/services/api/`
- **i18n:** inline `language === 'ru' ? '...' : '...'` (русский + узбекский)

Все правила проекта — в `CLAUDE.md` в корне. Соблюдай их строго: миграции при изменении схемы, мульти-тенантность через `tenant_id`, без `any` без необходимости, без дублирования логики.

## Правила реализации

1. **Сохраняем фирменную палитру:** primary остаётся `#F59E0B` (Tailwind `amber-500`). Дополнительные оттенки — `amber-50/200/400/600/700`, нейтрали `slate-50…900`, семантика — `emerald-500` (success), `red-500` (danger), `blue-500` (info).
2. **Mobile-first.** Жилец заходит с телефона. Все компоненты обязаны нормально работать на 375px ширине. Десктопная версия — это та же мобильная в центре экрана + sidebar (как сейчас).
3. **Bottom-nav на мобильном** (Главная / Заявки / Чат / Профиль) — добавить, на десктопе скрыть.
4. **Никакого шума.** Старая шапка с тикающим временем `18:11:29` — убрать. Фоновые soft-orange круги в углах — убрать.
5. **Платёжные системы и счётчики НЕ интегрированы** — реализуй их как карточку «Уже скоро · в разработке» (пунктирная рамка, серая, без CTA). Никаких заглушек с фейковыми долгами.

## Скоуп — 3 итерации

### P0a · Главная резидента (`/`)

Файл: `src/frontend/src/pages/resident/components/HomeTab.tsx`

Структура сверху вниз:

1. **Header** (sticky): «ЖК [имя] · Дом [номер]» (тон ink-400) + «Добрый вечер, [имя]» (h1) + bell-icon с бейджем непрочитанных. Под этим — pill «📍 ул. ..., кв. ..., NN м²».
2. **Карточка активного голосования** (если есть): badge «ВАЖНО» в углу, иконка ✓ слева, заголовок повестки, прогресс кворума, таймер до закрытия, CTA «Проголосовать →» (синий, full-width).
3. **Карточка активной заявки**: иконка категории + ID + название, pill статуса справа, прогресс из 4 шагов с подписями (Создана / Назначена / В работе / Завершена), внутренняя строка с исполнителем (avatar, имя, рейтинг, кол-во заявок) + круглая кнопка-звонок (зелёная).
4. **Карточка «Уже скоро»** (пунктирная рамка ink-200): два ряда — «Онлайн-оплата · Click и Payme» и «Передача показаний счётчиков». Каждая иконка в светло-серой плашке. Никаких CTA, только информирование.
5. **Быстрые действия** (grid 2×2): Создать заявку / QR гостю / Чат с УК / Оценить УК. Каждая — карточка с иконкой в плашке цвета фичи.
6. **Свежие объявления** (top-3): pill категории (Срочно/Инфо/Жильцы) + заголовок + время.
7. **Bottom nav** (только на мобильном).

Уровни данных: `useDataStore`/`useRequestStore`/`useMeetingStore`/`useAnnouncementStore` — точечные подписки на нужные поля, не на весь store.

### P0b · Создать заявку (wizard)

Файл: новый или рефактор `src/frontend/src/pages/resident/components/NewRequestModal.tsx`. Заменить на `<Sheet>`-флоу из 3 шагов:

**Шаг 1 — Категория:** grid категорий (сантехник, электрик, уборка, охрана, лифт, …) подгружается из `categories` API. На каждой иконка lucide + название + sub «среднее время N мин».

**Шаг 2 — Описание:**
- recap-карточка с выбранной категорией (cl. brand-50, кнопка «Изменить»)
- textarea (счётчик 0/500, кнопка «голосовая запись» снизу — заглушка пока, привязывать на будущее)
- блок фото: до 5 штук, превью с кнопкой удалить, последняя плитка — «+ Добавить»
- блок «Когда удобно»: grid 2×2 слотов. Сегодня до 21:00 (помечен как primary с галочкой), завтра 10-14, завтра 14-18, «выбрать в календаре». Слоты подгружаются из исполнительского расписания если такое API есть.
- toggle «Доступ в квартиру» (Кто-то будет дома / Передам ключ соседу)
- зелёная подсказка «Среднее время по дому: 2.5ч · 4.7★ от 142 жильцов» — реальные данные из `executor_ratings` для исполнителей этой категории в этом здании.

**Шаг 3 — Подтверждение:** свод всего + кнопка «Отправить заявку».

Sticky bottom-CTA «Далее →» / «Отправить →» — на мобилке всегда видна.

Step progress (3 палочки сверху, активные — brand-500, неактивные — ink-100).

### P1 · Голосование (`/meetings/:id/vote`)

Файл: создать `src/frontend/src/pages/MeetingVotingPage.tsx`, использовать существующий `meetingVotingStore.ts`.

Сверху вниз:

1. **Header**: back-кнопка + «Собрание собственников · [месяц]» + h1 «Голосование открыто», pills таймера и кворума.
2. **Hero карточка** (оранжевый градиент): «Ваш голос: NN м² = NN% площади дома». Просветительский текст про РУз.
3. **Кворум**: progress bar (две части: ok-500 от 0 до 50%, emerald-300 от 50% до текущего значения), подписи «Проголосовало: NNNN м² / Всего: NNNN м²».
4. **Каждый пункт повестки** — карточка с brand-200 рамкой (если активный):
   - pill «Пункт N из M» + pill с прикреплённым файлом (если есть)
   - заголовок и описание
   - блок «Промежуточные результаты» — 3 строки (За/Против/Воздержался) с горизонтальными прогрессами
   - 3 кнопки голоса (За/Против/Воздержусь). Выбранная подсвечивается ring + ok-50 фон.
   - Если выбран «Против» — раскрывается обязательное textarea «Причина возражения» (`comment_type='objection'`) + опциональное textarea «Контрпредложение» (`counter_proposal`).
5. **Тёмная OTP-карточка**: 6-cell grid для кода из SMS, кнопка «Отдать голос» (brand-500).

Бэк-логика:
- При голосовании первый раз — INSERT в `meeting_vote_records`. Повторное — UPDATE (UNIQUE constraint на `(meeting_id, agenda_item_id, voter_id)`).
- `vote_weight` = `users.total_area`.
- Комментарии — в `meeting_agenda_comments` (поля `resident_id`, `resident_name`, `content`).
- После каждого голоса — обновить агрегаты на фронте без перезагрузки.

## Технические требования

- Все новые UI-блоки — в `src/frontend/src/components/` (если переиспользуемые) или в папке экрана.
- При добавлении новых stores — отдельный файл, барреллингом через `dataStore.ts`/`crmStore.ts`.
- При мутациях бэка — `invalidateCache('key:')`.
- Без `useEffect` без dep array. Большие вычисления — `useMemo`/`useCallback`.
- Все запросы к БД — фильтр по `tenant_id`.
- Push-уведомления — `sendPushNotification(env, userId, {title, body, type, tag, data}).catch(() => {})`.

2. `npm run build` — собирается.
3. Открыть в Chrome на эмуляции iPhone 14 (DevTools → Toggle Device Toolbar) — все 3 экрана выглядят как в `redesign/kamizo-resident-redesign.html`.
