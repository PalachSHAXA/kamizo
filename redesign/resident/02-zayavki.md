# 02 — Заявки жителя (Requests tab)

**Компонент:** `pages/ResidentDashboard.tsx` (вкладка `requests`) + `pages/resident/components/RequestsTab.tsx`,
карточки `HistoryRequestCard.tsx`, `PendingApprovalCard.tsx`.

## Что делает экран (функционал)

Список заявок жителя (`/?tab=requests`, открывается из BottomBar). Реальные фичи:

- **Под-вкладки** (`requestsSubTab`): Активные / На приёмке (pending_approval) / История
  (completed + cancelled, отсортированы по дате).
- **Фильтры** (`showFilters`): по категории (специализация) и приоритету.
- Статусы заявки: new, assigned, accepted, in_progress, pending_approval, completed, cancelled.
- Карточка заявки: номер `#`, тип с иконкой, адрес, исполнитель, дата, бейдж статуса, фото-превью.
- Тап по карточке → RequestDetailsModal (см. файл 04). Кнопка приёмки → ApproveModal.
- Кнопка «Новая заявка» открывает ServiceBottomSheet (см. файл 03).
- Состояние загрузки (`isLoadingRequests`) и пустое (нет заявок).

## Креативное направление

Чистая лента-таймлайн. Сегмент-таб-бар (Активные/Приёмка/История) на `--surface-sunken`.
Карточки заявок — белые `--radius-lg`, слева цветная вертикальная метка статуса (тонкая, из
семантических токенов), тип-иконка в кружке, справа бейдж статуса. Срочные — критическая метка +
бейдж «Срочно» (`--brand-tint`/critical). У активных — мини-прогресс стадии (Принята → В работе →
Готово) тонкой шкалой. Фото-превью — скруглённые thumbnail-стрип.

**Анимации:** смена под-вкладки — плавный кроссфейд + сдвиг индикатора; новые заявки въезжают
сверху; статус-переходы подсвечиваются; фильтр-панель — выезд-аккордеон.

## Состояния

- Загрузка: 3–4 скелетон-карточки.
- Пусто (нет заявок): иллюстрация + «Заявок пока нет» + кнопка «Создать заявку».
- Фильтр без результатов: «Ничего не найдено, сбросить фильтр».
