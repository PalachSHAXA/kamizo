# Kamizo — Промты для редизайна по страницам

> **Как фетчить:** для каждой страницы (1) вставь сначала `00-FOUNDATION.md`, затем (2) промт нужной
> страницы отсюда. Claude вернёт перерисованный компонент. Делай по одному экрану за раз — так
> легче ревьюить и не сломать роутинг.
>
> Каждый промт уже предполагает, что Claude знает палитру/токены из FOUNDATION — не дублируй их.

## Группа A — Вход и навигация

### A1. Страница входа — `pages/LoginPage.tsx`
```
Используй дизайн-систему из FOUNDATION (бежевый + оранж + тёмный stone).

Перерисуй экран входа Kamizo. Это первое впечатление о per-tenant УК.

Композиция (desktop — две колонки, mobile — одна):
- Левая/верхняя колонка: тёплый бежевый фон (--app-bg), по центру карточка входа на --surface,
  rounded-[--radius-xl], shadow-md. Логотип тенанта сверху, заголовок «Вход в Kamizo»,
  поле телефон/логин + пароль, первичная оранжевая кнопка на всю ширину, ссылка-помощь снизу.

  фразой о сервисе и тонкой бежевой типографикой, без стокового фото.
Состояния: ошибка входа (статус critical плашка), загрузка (спиннер в кнопке), фокус инпутов.
i18n RU/UZ. Сохрани логику авторизации и существующие хендлеры.
```

### A2. Навигация — `components/layout/Sidebar.tsx`, `Header.tsx`, `MobileHeader.tsx`, `components/BottomBar.tsx`
```
Используй дизайн-систему из FOUNDATION.

Перерисуй навигацию (4 компонента, по очереди — начни с Sidebar):
- Sidebar (desktop): тёмный stone-фон (--dark-surface), пункты с lucide-иконкой 20px + текст,
```
