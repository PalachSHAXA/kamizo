## 4. БЕЗОПАСНОСТЬ

### 4.1. JWT ✅ БЕЗОПАСНО

- **Алгоритм:** HMAC-SHA256 (Web Crypto API)
- **Структура:** Полноценный 3-частный JWT (header.payload.signature)
- **Expiration:** 24 часа, проверяется при каждом запросе
- **Fallback на raw token:** УДАЛЁН (2026-03-16)
- **Файл:** `cloudflare/src/utils/crypto.ts`, строки 42-82

### 4.2. Security Headers ✅ ПОЛНЫЙ НАБОР

| Header | Значение | Статус |
|--------|----------|--------|
| X-Frame-Options | DENY | ✅ |
| X-Content-Type-Options | nosniff | ✅ |
| Strict-Transport-Security | max-age=31536000; includeSubDomains | ✅ |
| Referrer-Policy | strict-origin-when-cross-origin | ✅ |
| Permissions-Policy | camera=(), microphone=(), geolocation=() | ✅ |
| Content-Security-Policy | Полная политика с frame-ancestors 'none' | ✅ |

### 4.3. SQL Injection ✅ ЗАЩИЩЕНО

**0 уязвимостей.** Все SQL-запросы используют параметризованные `.bind()` вызовы. Template literals используются только для условий WHERE (tenant_id), но не для пользовательского ввода.

### 4.4. Rate Limiting ✅ FAIL-CLOSED

- Login: 5 запросов/мин
- Register: 60 запросов/мин
- Default: 100 запросов/мин
- При ошибке KV: запрос **блокируется** (fail-closed)

### 4.5. CORS ✅ для Production

- Production: только `*.kamizo.uz` (3 домена + динамические поддомены)
- Dev: localhost:5173 и :3000 при `ENVIRONMENT !== 'production'`
- `.dev.vars` в `.gitignore` ✅

### 4.6. Шифрование ⚠️

- AES-GCM с 12-байтным случайным IV ✅
- PBKDF2 с SHA-256, 50,000 итераций для паролей ✅
- ⚠️ Нет механизма ротации ключей

### 4.7. WebSocket 🔴 КРИТИЧНО

**Файл:** `cloudflare/src/ConnectionManager.ts`, строка 86

```
const userId = url.searchParams.get('userId');
```

**Проблема:** Аутентификация через URL-параметры без верификации JWT. Клиент может подставить любой userId/role. Tenant isolation реализована (строка 554), но бесполезна при подделке userId.

### 4.8. Hardcoded Secrets 🔴 КРИТИЧНО

| Секрет | Файл | Проблема |
|--------|------|----------|
| VAPID_PRIVATE_KEY | `notifications.ts:13` | Приватный ключ в исходном коде |
| ENCRYPTION_KEY | `.dev.vars:1` | В .gitignore, но слабый |
| JWT_SECRET | `.dev.vars:2` | В .gitignore, 22 символа |

**VAPID приватный ключ ДОЛЖЕН быть перенесён в `wrangler secret`.**

### 4.9. Feature Gating ✅

- **279 проверок** requireFeature() по всей кодовой базе
- Все крупные модули защищены
- auth.ts, users.ts, super-admin.ts корректно исключены (базовый/ролевой доступ)
